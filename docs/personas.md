# Hackathon Persona 解释与计分边界

> 状态：人格语义基线
> 更新日期：2026-07-21
> 用途：指导问卷出题、选项赋分、结果文案和 Pet 视觉设计

## 使用原则

- 这 8 个人格描述的是参赛者在 Hackathon 里的行为倾向，不是对现实身份的诊断。其中 `嘉豪` 是有意保留负面讽刺的人格，讽刺对象是具体的装懂行为，而不是现实中使用这个名字的人。
- 题目应观察具体行为，例如如何组队、选题、开发、救火和路演；不要直接让用户选择人格标签。
- 一个选项可以同时给多个人格加分，但高分必须对应明确的行为证据。
- 人格覆盖看整套问卷，不看单道题。先写现场中真实可能发生的选择，再给有证据的人格赋分；允许一行只有一两列得分，不得为了照顾某个人格硬造不自然的选项。
- 每个保留选项至少要给一个人格提供正分证据。全零行没有分类信息，应删除该选项而不是硬贴人格；如果某类自然行为反复无法映射，应重新检查人格集合。
- 每道题没有目标选项数，只保留真实、彼此有区别的应对方式。`n ≥ 2` 是数据结构下限，不是默认要写满 4、5 或 6 项；主要用于填补人格列的选项应直接删除。
- 选项不能全部写成体面的团队最佳实践。场景允许时，可以出现偏袒、逃避、自私或不负责任但现实中确实会发生的选择；不要求每题机械配置一个“坏选项”。
- `0` 分表示该选项没有提供相关证据，不表示反对或负面评价。
- 代码 ID 是稳定标识。中文名、英文展示名和结果文案可以迭代，ID 不应随文案变化。
- `魔丸`、`💩` 等夸张人格是娱乐化结果。题目和结果文案不能把破坏团队、骚扰他人或不卫生行为包装成值得模仿的做法。

## 容易混淆的人格

| 人格组合 | 关键区别 | 出题时的区分方法 |
| --- | --- | --- |
| 科比牢大 vs. 耄耋 | 科比牢大在压力下把注意力放在“我来做完”；耄耋在感到被冒犯时把注意力放在“你是不是针对我”。 | 接管任务、用结果解决问题偏科比牢大；因语气、反驳或忽视而急眼并攻击别人，才偏耄耋。 |
| 大狗叫 vs. 嘉豪 | 区别不在音量，而在表达有没有真实内容。大狗叫再吵也在讲项目、招人或推动路演；嘉豪用高端名词掩盖“不懂、不会实现”。 | 能用具体功能、用户价值或实现进度回答追问，即使表达夸张也偏大狗叫；只讲宏大概念，追问落地便换词，才偏嘉豪。 |
| 咕咕嘎嘎 vs. 奶龙大笑 | 咕咕嘎嘎主要搬运内容、经营群聊；奶龙大笑主要用即时情绪反应影响现场。 | 发视频、造内部梗、刷群聊偏咕咕嘎嘎；当场捧腹、破冰和笑着救场偏奶龙大笑。 |
| 魔丸 vs. 💩 | 魔丸代表高风险操作和技术事故；💩只代表真实的物理排泄。两者不再共享“制造混乱”这一抽象信号。 | 强推、关服务、改环境、制造事故只给魔丸；只有找厕所、上厕所或随地大小便才给 💩。 |

## 计分列顺序

每个选项的分数行必须按照以下顺序写满 8 列：

```text
kobe-laoda → maodie → big-dog-bark → gugugaga
→ mowan → jiahao → poop → nailong-daxiao
```

对应中文顺序：

```text
科比牢大 → 耄耋 → 大狗叫 → 咕咕嘎嘎
→ 魔丸 → 嘉豪 → 💩 → 奶龙大笑
```

分数同时表达行为证据和人格覆盖稀缺度，不要求所有人格共用同一个最高分：

| 分数 | 含义 |
| ---: | --- |
| 10 | 仅用于 💩 的两个物理排泄彩蛋选项。 |
| 5–6 | 覆盖较少人格（如咕咕嘎嘎、魔丸、嘉豪）的直接核心信号。 |
| 3–4 | 普通人格的强信号，或覆盖较少人格的次强信号。 |
| 2 | 覆盖较广人格（尤其科比牢大）的直接信号，或其他人格的次级关联。 |
| 1 | 只有轻微信号。 |
| 0 | 没有足够证据。 |

每次调整矩阵后，都要穷举全部答案组合。除 💩 外，七个人格在等概率选项假设下的最终结果占比最大差距不得超过 5 个百分点。这个指标只用于检查矩阵结构，不能替代真实用户数据校准。

最终语义和代码 ID 以本文档与 `src/personas.ts` 的一致内容为准；实际题干、选项和矩阵配置位于 `src/questionnaire.json`，读取与校验逻辑位于 `src/questionnaire.ts`。
